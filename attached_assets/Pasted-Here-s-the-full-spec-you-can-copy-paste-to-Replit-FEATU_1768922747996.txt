FEATURE SPEC: LogicProcess + CoWork for Orchestrate
Overview
Add workflow building and running capabilities to Orchestrate. This is TWO MODES of the same tool:

LogicProcess (Build Mode) - Developers create workflow templates
CoWork (Run Mode) - End users run workflows with their own data

Architecture
Orchestrate (orchestrate.us.com) is ONE Replit app that includes:

Control Room (existing)
LogicProcess (new - Build Mode)
CoWork (new - Run Mode)

It integrates with existing APIs:

Quack (quack.us.com) for agent messaging
LogicArt (logic.art) for code visualization

Execution Engine
When a workflow runs:

Start - User clicks "Run Workflow"
Collect inputs - Gather all runtime inputs from user
Execute nodes - For each node:

If AI Agent: Send Quack message, wait for reply
If Transform: Process data locally
If Human Review: Pause, show UI, wait for approval
If Condition: Evaluate and branch


Pass data - Output from each node feeds into next
Complete - Show final results, offer download

Quack Integration
Each AI Agent node sends a Quack message:
jsonPOST https://quack.us.com/api/send
{
  "to": "claude",
  "from": "orchestrate",
  "task": "Generate blog outlines for each topic",
  "context": "Workflow: Content Pipeline, Node: Generate Outlines",
  "files": [
    { "name": "topics.csv", "content": "..." }
  ],
  "replyTo": "workflow-run-123"
}
```

When reply received, workflow continues to next node.
```
